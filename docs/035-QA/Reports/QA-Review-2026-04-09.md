# QA Review Report — April 2026

## Executive Summary

150 E2E spec files across 19 domains were reviewed by a 6-agent parallel tester team. The review found **4 test files that never execute** (broken imports), **12+ tests that always pass regardless of application behavior** (false positives via conditional guards), **6 assertions encoding server crashes as expected behavior** (HTTP 500 for business-rule violations), and **3 tautological assertions** that can never fail. Additionally, ~40 test cases run 2-3× per CI run due to pervasive file-level duplication. Critical security tests — rate limiting, CSRF protection, file upload — provide **zero actual coverage** despite appearing in reports.

## Silent False Positives via `if (isVisible)` Guards

12+ tests wrap their **entire assertion body** inside `if (await element.isVisible())`. When the element is absent (seed failure, UI bug, feature flag), the test reports ✅ green with **zero assertions executed**.

Affected files:
- `customers/create.spec.ts` — entire create flow conditional
- `customers/deactivate.spec.ts` — entire deactivation conditional; **state restore NOT in afterEach**
- `users/update.spec.ts` — role badge assertion conditional
- `users/search.spec.ts` + `customers/search.spec.ts` — assert URL contains "search", not result content
- `guest/pagination.spec.ts` — both assertion blocks conditional
- `guest/search-filter.spec.ts` — all three interaction blocks conditional
- `store/product-detail.spec.ts` — falls back to "any first product" when seeded product not found
- `dashboard/date-range.spec.ts` — both test bodies entirely conditional
- `dashboard/comparison.spec.ts` — only assertion conditional
- `dashboard/widgets.spec.ts` TC-DASH-008 — only assertion conditional
- `accounting/finance-ui.spec.ts` — 6 separate tests all guarded

## Coverage Gaps (Not Tested Anywhere)

- **Cart and checkout flow** — no test for add-to-cart, update quantity, checkout
- **Full order lifecycle** — PENDING → PAID → SHIPPING → DELIVERED never in one test
- **Session fixation prevention** — no test that session token changes after login
- **Session/token expiry** — no test that expired cookie is rejected
- **Card payment method** — `PAYMENT_METHOD.CARD` from constants never used in any test
- **Expense edit and delete flows** — completely absent from E2E
- **Customer profile edit** — update name/phone/address never tested
- **Customer reactivation cycle** — deactivate → reactivate lifecycle not tested
- **Inactive product hidden from storefront** — `TEST_PRODUCTS.inactive` fixture exists, never tested
- **Product sort order** — `PRODUCT_SORT` constants defined, never exercised
- **Storefront accessibility** — all 8 a11y tests target admin routes only; guest storefront has zero axe coverage
- **Database-level RLS** — `rls-policy.spec.ts` only tests HTTP RBAC; no row-level isolation
- **Concurrent session handling** — same user in two browsers simultaneously
- **HTTP method override attacks** — `X-HTTP-Method-Override` header
- **Real-time stock depletion sync** — price sync tested; stock depletion not
- **Mobile layout** — Playwright config targets Mobile Chrome/Safari; zero mobile tests
