# TC-A11Y-001: Storefront Accessibility Scan

## Pre-conditions

- Storefront is running and accessible.
- axe-core / `@axe-core/playwright` installed.

## Test Steps

1. Navigate to the storefront home page (guest context).
2. Run `checkA11y` with tags `["wcag2a", "wcag2aa", "wcag21aa"]`.
3. Navigate to the product listing page.
4. Run `checkA11y`.
5. Navigate to a product detail page.
6. Run `checkA11y`.

## Expected Result

- Zero WCAG 2.1 AA violations on all three storefront pages.

## Spec File

`tests/e2e/a11y-storefront.spec.ts` *(to be created)*

## Review Status

🔴 Not Tested

## Review Findings

- All 8 existing `a11y.spec.ts` tests cover **admin routes only** (`/login`, `/dashboard`, `/products`, `/customers`, etc.).
- The customer-facing storefront — accessible to users with disabilities — has **zero accessibility coverage**.
- Admin and storefront use different UI components; admin a11y coverage does not imply storefront compliance.

## Coverage Gaps

- Storefront home, listing, product detail
- Interactive states: variant selection, search input focus
- Mobile viewport accessibility
